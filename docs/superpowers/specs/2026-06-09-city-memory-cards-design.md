## 目标

在“城市印象 → 城市记忆”中，将每个县级区划的第一人称旅游自我介绍以更“古色古香”的卡片化形式呈现：层次清晰、图文并茂、阅读节奏更像一页游记。

## 数据来源与生成

- 每个县域的“城市记忆”正文由 DeepSeek 动态生成并缓存到 localStorage（已实现）。
- 无 Key 时展示简版模板，避免空白（已实现）。
- 本次变更仅调整“展示与排版”，不改变生成与缓存策略。

## 展示结构（卡片拆分）

将一篇 markdown 内容按章节拆为卡片：

- 封面卡：来自 `#` 一级标题与其后紧随的开场段落（直到遇到第一个 `##`）
- 正文卡：每个 `##` 二级标题对应一张卡
  - `###` 三级标题作为卡内的小节标题
  - `-` 列表保留为卡内要点

## 视觉风格（古色古香）

卡片整体风格参考“宣纸 + 朱砂印 + 墨色”：

- 背景：半透明纸感底色，轻微纹理/渐变
- 边框：虚线或双线边框，模拟装帧
- 阴影：极浅，模拟纸张叠放
- 字体：沿用现有宋体/楷体风格（Noto Serif SC / ZCOOL XiaoWei），标题更重，正文更松
- 装饰：卡片左上角“朱砂小印”式图标底座（圆形/方章），内部用内置 SVG 图标

## 图标策略

- 不引入外部依赖库
- 使用内置 SVG（inline）以保证一致的墨色与缩放效果
- `##` 章节到图标的映射（可扩展）：
  - 我的山水家底：山/水纹
  - 我的千年故事：卷轴/塔
  - 我的文墨风华：毛笔/书卷
  - 我的生活日常：茶盏/烟火
  - 怎么找到我：罗盘/足迹
  - 未匹配：默认“印”图形

## 渲染与安全

- 继续保持“安全渲染”，不直接注入用户/模型输出的 HTML
- markdown 支持的子集：
  - `#` `##` `###`
  - 段落文本
  - `-` 无序列表
  - `**加粗**`
- 渲染流程：
  1. 将 markdown 解析为结构化 sections（cover + cards）
  2. 对每个文本节点做 HTML escape
  3. 仅由代码生成安全的 HTML 标签（卡片、标题、列表、段落）

## 交互与状态

- 打开侧栏后立即渲染：若 memory 已存在则展示卡片；若在生成中则显示“生成中…”占位卡
- 生成完成自动刷新当前卡片内容
- 不新增额外按钮（保持阅读沉浸）

## 验收标准

- “城市记忆”展示为多张卡片，封面卡 + 多张章节卡
- 卡片具备古色古香样式，图标与标题对齐，阅读清晰
- 不出现 XSS 风险：模型输出的内容不会以 HTML 执行
- 与现有侧边栏滚动、切换 Tab 不冲突
