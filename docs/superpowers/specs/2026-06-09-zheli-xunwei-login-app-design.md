## 项目目标

为“浙里寻味·美食寻宝图”实现可直接运行的生产级前端工程（React 18 + TypeScript + Vite + Tailwind CSS v3 + React Router v6），覆盖：

- 登录流程（/login）
- 登录后加载过渡页（/loading，1.5s）
- 主界面（/：左侧导航栏 + 右侧地图）
- 4 个核心子页面（打卡/收藏/日记/设置），以右侧滑入抽屉覆盖地图 80%

设计风格严格遵循“江南烟火气 × 现代寻宝感”，并精确使用指定色值。

## 非目标

- 不接入真实地图 API，不做省市县真实 GIS 渲染，仅用 CSS/SVG 做简化浙江地图示意
- 不接入真实后端接口，仅提供完整模拟数据与本地登录态
- 不引入任何额外 UI 组件库与图标库（仅 lucide-react）

## 技术栈与约束

- React 18 + TypeScript
- Vite
- Tailwind CSS v3（JIT）
- react-router-dom v6
- 状态：React Context API
- 图标：lucide-react（允许保留用户指定的 emoji 菜单前缀）
- 语义化 HTML5 元素

## 路由与页面结构

采用“路由驱动抽屉子页”的结构，保证：

- 支持浏览器前进/后退
- 子页面可深链
- “点击地图空白处关闭子页面”可映射为 navigate 回到 /

路由定义：

- /login：登录页
- /loading：登录后加载过渡页（1.5 秒后跳转到 /）
- /：App Layout（包含 Sidebar + Map 作为底层）
  - index：Map 主视图（地图与浮层控件）
  - /checkins：右侧抽屉 - 我的打卡
  - /favorites：右侧抽屉 - 我的收藏
  - /diary：右侧抽屉 - 美食日记
  - /settings：右侧抽屉 - 账号设置

路由守卫：

- 未登录访问 / 与抽屉子页，重定向到 /login
- 已登录访问 /login，重定向到 /

## 全局状态（Context）

### AuthContext

职责：

- 管理 user（模拟用户信息）与登录态
- 提供 login/logout 方法
- 本地持久化：localStorage（默认记住登录，贴近真实产品）

数据键（建议）：

- zlxw_auth_v1：{ token: string; user: User; }

### ToastContext

职责：

- 轻量 Toast 队列：显示成功/失败提示
- 登录成功 Toast：欢迎回来，美食探险家（1.5s 后跳转）

### UIContext（可选）

职责：

- 侧栏折叠态（desktop 可手动折叠；tablet 默认折叠）
- 移动端底部 Tab 当前态（也可从路由派生）

## 登录页（/login）规格

### 布局

- 桌面端：左右分栏 6:4
- 移动端：上下布局

### 左侧品牌区（定稿：方案 A + 点缀宝箱 + 浙江风味水墨）

视觉：

- 背景：淡墨山水底纹（多层 radial/linear gradient 模拟）+ 半透明暖杏黄滤镜（--warm-apricot）
- 顶部：书法字体“浙里寻味” + 青瓷印章装饰（--primary 低透明）
- 中部：青瓷宝箱（点缀感：偏小，位于中部偏下，不抢主视觉）
- 飞出图标：3–4 个“江南烟火通用”简笔画符号（使用 lucide-react 可用的生活器物图标组合），不同延迟与轨迹
- 左下：乌篷船缓慢漂移
- 右上：蒸汽效果浮动
- 底部 slogan：一张地图，吃遍浙江百县千味（--sauce-red）

动效：

- 宝箱轻微呼吸/浮动（默认）
- 触发（点击宝箱或图标）：宝箱打开 + 金色光芒 + 星星粒子 + 图标飞出（CSS keyframes）

### 右侧登录区

容器：

- 白色卡片，rounded-2xl，shadow-sm，padding 40px

表单：

- 标题：开启你的浙江美食寻宝之旅（text-xl font-bold text-gray-800）
- 手机号输入框
  - 左侧手机图标（lucide-react）
  - 输入时自动格式化：3-4-4
  - focus 边框切到 --primary
- 验证码输入框
  - 左侧锁图标（lucide-react）
  - 右侧“获取验证码”按钮：点击后 60s 倒计时禁用
- 登录按钮
  - 全宽
  - 背景 --primary，hover --primary-dark
  - active 有轻微下沉/缩放
- 底部用户协议链接（text-xs，--text-muted）

校验与错误提示：

- 手机号：11 位数字（以格式化后的纯数字为准）
- 验证码：6 位数字
- 错误提示：--sauce-red，显示在对应输入框下方

成功流程：

- 通过校验后视为登录成功（模拟）
- Toast：“欢迎回来，美食探险家”
- 1.5 秒后 navigate 到 /loading

## 加载过渡页（/loading）规格

- 全屏淡墨山水背景（与登录页风格一致但更干净）
- 中心动画：宝箱打开 → 金色光芒迸发 → 美食图标四散 → 显示“浙里寻味”logo
- 文案：正在加载美食宝藏...
- 时长：1.5 秒，结束后 navigate 到 /

## 主界面（/）规格

### 总体布局

- 左侧固定导航栏（240px） + 右侧地图主区域（撑满）
- 侧栏折叠：折叠到 60px，仅显示图标（宽度平滑过渡）

### 侧栏（Sidebar）

视觉：

- 背景：--primary，opacity 95%，height 100vh
- 用户区：头像（青瓷色背景 + 白色碗筷图标）、昵称、等级
- 版权：浙里寻味 © 2026（text-xs text-white/60）

菜单项（4 个）：

- 📍 我的打卡（/checkins）
- ❤️ 我的收藏（/favorites）
- 📝 美食日记（/diary）
- ⚙️ 账号设置（/settings）

交互状态：

- 默认：白色
- hover：背景加深 + 左侧 3px --sauce-red 竖线
- active：--sauce-red 背景 + 白字 + 左侧 3px 白竖线

### 地图主区域（Map）

视觉：

- 背景：--warm-apricot + 淡墨浙江轮廓底纹

地图元素（示意）：

- 县市节点：显示名称
- 未打卡：灰色宝箱图标
- 已打卡：彩色美食图标 + 浅酱红背景
- hover：弹出预览卡（名称 + 一句 slogan）

浮层控件：

- 搜索栏（右上）：白色背景、圆角 8px、搜索图标 + 输入 + 筛选按钮
- 地图控制区（右下）：放大/缩小、定位、全屏按钮

## 子页面抽屉（滑入式）规格

通用规则：

- 从右侧滑入，默认覆盖 80% 地图区域
- 遮罩层点击关闭：navigate 回到 /
- 动效：滑入滑出 + 轻微淡入淡出

响应式：

- 平板端（768–1199）：抽屉改为全屏覆盖
- 移动端（<768）：页面全屏显示，主导航切换为底部 Tab

## 四个子页面内容规格（概要）

### 📍 我的打卡

- 标题：我的美食打卡（书法字体，--primary）
- 统计概览：3 个统计卡 + 总进度条
- 双栏：左 40% 迷你浙江地图（已打卡酱红高亮）；右 60% 分组列表（图片/名称/店铺/时间）
- 底部：成就徽章墙（已获得/未获得）

### ❤️ 我的收藏

- 标题：我的收藏
- 标签栏：全部 | 店铺 | 美食 | 地区 | 帖子
- 内容：瀑布流卡片（桌面每行 3 个）
- 空状态：还没有收藏任何内容 + 去探索按钮

### 📝 美食日记

- 标题：我的美食日记
- 切换：我的作品 | 我的评论
- 作品：瀑布流（图文 + 点赞评论数）
- 悬浮“+”：进入发布页（可作为抽屉内二级视图或独立路由）
- 评论：时间倒序列表

### ⚙️ 账号设置

- 标题：账号设置
- 左侧菜单：个人信息 / 修改密码 / 通知设置 / 隐私设置 / 关于我们 / 退出登录
- 右侧默认个人信息：头像更换、昵称编辑、手机号脱敏显示 + 更换、简介编辑、保存按钮

## 动效与音效（全局）

- 页面转场：淡入淡出 + 轻微滑动
- 按钮反馈：hover 变色，active 轻微缩放
- 宝箱动画：点击美食图标触发宝箱打开 + 星星/香气特效（CSS）
- 打卡成功音效：Web Audio API 播放清脆“叮”（无需外部音频文件）

## 模拟数据

至少包含：

- 用户信息：昵称、等级、头像（默认）
- 打卡数据：县市、对应美食、店铺、时间、已获得徽章
- 收藏数据：类型（店铺/美食/地区/帖子）、封面图（占位图策略）、收藏时间
- 日记数据：作品（图/文/赞评数）、评论（内容/关联帖子/时间）
- 地图节点：县市列表、是否打卡、hover 预览信息

## 错误边界与空状态

- 全局 ErrorBoundary 捕获渲染错误，展示温和的纸感错误页与“返回首页”
- 子页面列表为空时展示空状态

## 交付验收

- 项目可通过 npm install && npm run dev 启动
- /login 完整表单校验 + 倒计时 + Toast + 跳转
- /loading 1.5 秒动效后自动进入 /
- / 主界面侧栏折叠/展开动效可用
- /checkins /favorites /diary /settings 以抽屉滑入形式呈现，点击遮罩关闭
- 响应式：桌面/平板/移动端行为符合约束
