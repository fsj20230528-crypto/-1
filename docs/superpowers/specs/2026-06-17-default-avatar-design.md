# 默认头像与可更换头像设计

## 背景

当前登录流程要求用户先上传头像，否则无法登录。用户希望：

- 登录时默认使用指定图片作为头像
- 用户可以自行决定是否更换头像
- 登录页、个人中心和右上角头像显示保持一致

## 目标

- 首次进入登录弹窗时，直接显示内置默认头像
- 用户不上传头像也可以完成登录
- 用户上传新头像后，后续统一显示用户自定义头像
- 保持现有手机号、昵称校验逻辑不变

## 方案

### 头像来源优先级

统一采用以下优先级：

1. 用户本地已保存的自定义头像
2. 内置默认头像
3. 兜底 SVG 图标（仅在默认头像资源不可用时使用）

### 交互调整

- 登录弹窗头像预览初始显示内置默认头像
- 登录弹窗中的按钮文案由“上传头像”改为“更换头像”
- 个人中心继续保留“更换头像”入口
- 右上角个人头像按钮与登录/个人中心预览保持同一头像来源

### 数据与校验

- 将默认头像作为运行时默认值注入，不强制写入本地存储
- 用户登录保存资料时，若未主动更换头像，则保存默认头像数据
- 去掉“请先上传头像”的必填限制
- 手机号与昵称校验规则保持不变

## 实现范围

- 调整 `index.html` 中登录头像初始化逻辑
- 调整头像预览与右上角头像同步逻辑
- 调整登录与资料保存时的默认头像回填逻辑
- 调整登录按钮附近文案

## 验收标准

- 未上传头像时，登录弹窗能直接看到默认头像
- 用户仅填写手机号和昵称即可成功登录
- 登录后右上角显示默认头像
- 在个人中心更换头像后，右上角、登录弹窗、个人中心预览都同步为新头像
